## Problem Overview
在 **UPLR2 → UPLR2.3 OS patch load** 过程中（microcode 405→407），系统报发 **MCE（Machine Check Exception）**。MCA dump 显示：
- **MLC bank uncorrected error**
- **PCC bit set**
- **Socket0/Socket1 compute cores 中 overflow**

该问题严重影响 **MSFT** 的操作系统补丁部署流程。

## Root Cause
**PCode** 在 ACode 中禁用了 **AD（Auto-Demotion）算法**，导致 ACode 在执行补丁后的 microcode 时错误地抛出 MCA 错误。

具体机制：PCode 关闭了 core auto-demotion，但 ACode 不知道这一变化，仍按旧的 demotion 逻辑运行，最终在特定执行路径上触发 memory error（MLC bank uncorrected）。

## Fix Description
解决方案通过 **TELEM_PCODE_2 register** 建立通信机制：
- PCode 通过该寄存器向软件及 ACode 通知 **auto-demotion 算法是否启用**
- 确保在 patch load 期间，ACode 与 PCode 同步掌握 demotion 状态
- 避免状态不一致导致的 MCA 错误

交付状态：修复已纳入后续 UPLR 版本（具体版本号按 GK approval 决策）。
